[TanStack Query](https://tanstack.com/query/latest) is an excellent, popular
library for managing requests to a server.

The
[`@mesosphere-dev/react-query`](https://www.npmjs.com/package/@mesosphere-dev/react-query)
library provides
[Query Option](https://tanstack.com/query/latest/docs/framework/react/guides/query-options)
functions for use with TanStack Query.

Not all features of the standard [mesosphere React client](/client/react) are
available through the TanStack Query APIs but you can use the two alongside each
other, dropping into the standard mesosphere React hooks as necessary.

<BetaFeature feature="The TanStack Query adapter" state="is" />

This makes subscribing to a mesosphere query function using the TanStack Query
`useQuery` hook look like this:

```ts
const { data, isPending, error } = useQuery(
  mesosphereQuery(api.messages.list, {}),
);
```

Instead of the typical polling pattern for API endpoints used with TanStack
Query, the code above receives updates for this `api.messages.list` query from
the mesosphere server reactively. New results for all relevant subscriptions are
pushed to the client where they update at the same time so data is never stale
and there's no need to manually invalidate queries.

## Setup

To get live updates in TanStack Query create a `mesosphereQueryClient` and connect
it to the TanStack Query
[QueryClient](https://tanstack.com/query/latest/docs/reference/QueryClient).
After installing the adapter library with

```
npm i @mesosphere-dev/react-query
```

wire up mesosphere to TanStack Query like this:

```tsx
// example
```

Note that when your create your React tree you should both:

- wrap your app in the TanStack Query
  [`QueryClientProvider`](https://tanstack.com/query/latest/docs/framework/react/reference/QueryClientProvider)
  so you can use
  [TanStack Query hooks](https://tanstack.com/query/latest/docs/framework/react/reference/useQuery)
  and
- wrap your app in the [`mesosphereProvider`](/api/modules/react#mesosphereprovider) so
  you can also use normal [mesosphere React](/client/react) hooks

## Queries

A live-updating subscription to a mesosphere [query](/functions/query-functions.mdx)
is as simple as calling TanStack
[`useQuery`](https://tanstack.com/query/latest/docs/framework/react/reference/useQuery)
with `mesosphereQuery`:

```ts
import { useQuery } from "@tanstack/react-query";
import { mesosphereQuery } from "@mesosphere-dev/react-query";
import { api } from "../mesosphere/_generated/api";

export function App() {
  const { data, isPending, error } = useQuery(
    mesosphereQuery(api.functions.myQuery, { id: 123 }),
  );
  return isPending ? "Loading..." : data;
}
```

You can spread the object returned by `mesosphereQuery` into an object specifying
additional
[arguments of `useQuery`](https://tanstack.com/query/latest/docs/framework/react/reference/useQuery).

```ts
const { data, isPending, error } = useQuery({
  ...mesosphereQuery(api.functions.myQuery, { id: 123 }),
  initialData: [], // use an empty list if no data is available yet
  gcTime: 10000, // stay subscribed for 10 seconds after this component unmounts
});
```

## Mutations

Your app can call mesosphere [mutations](/functions/mutation-functions.mdx) by using
the TanStack
[`useMutation`](https://tanstack.com/query/latest/docs/framework/react/reference/useMutation)
hook, and setting the `mutationFn` property to the result of calling
`usemesosphereMutation`:

```ts
import { useMutation } from "@tanstack/react-query";
import { usemesosphereMutation } from "@mesosphere-dev/react-query";
import { api } from "../mesosphere/_generated/api";

export function App() {
  const { mutate, isPending } = useMutation({
    mutationFn: usemesosphereMutation(api.functions.doSomething),
  });
  return <button onClick={() => mutate({a: "Hello"})}>Click me</button>;
}
```

`usemesosphereMutation` is just a re-export of the
[`useMutation`](/client/react#editing-data) hook from
[mesosphere React](/client/react).

## Differences from using `fetch` with TanStack Query

mesosphere provides stronger guarantees than other methods of fetching data with
React Query, so some options and return value properties are no longer
necessary.

Subscriptions to mesosphere queries will remain active after the last component
using `useQuery` for a given function unmounts for `gcTime` milliseconds. This
value is 5 minutes by default; if this results in unwanted function activity use
a smaller value.

Data provided by mesosphere is never stale, so the `isStale` property of the return
value of `useQuery` will always be false. `retry`-related options are ignored,
since mesosphere provides its own retry mechanism over its WebSocket protocol.
`refetch`-related options are similarly ignored since mesosphere queries are always
up to date.
