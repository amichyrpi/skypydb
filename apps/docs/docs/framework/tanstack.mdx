To get up and running with Mesosphere in a TanStack Start project, run the following command:

```shell shell
npm create mesosphere@latest -- --template tanstack-start
```

or follow the guide below.

Learn how to use read and write functions in your TanStack Start app.

1. **Create a TanStack Start app** using create start app command.

```shell shell
npx create-start-app@latest
```

2. **After creating your app**, install the Mesosphere package:

```shell shell
npm install mesosphere
```

3. **Start the Mesosphere dev server** in a separate terminal window, it will automatically create a `mesosphere/` folder containing a `deploy.ts` file and a `.env` file in your project's root directory, don't forget to keep the command (`npx mesosphere dev`) running in the background. It will automatically update your functions when you make changes:

```shell shell
npx mesosphere dev
```

4. **Create a write and read functions** in the `mesosphere/` folder. For example, create a file called `tasks.ts` and add the following code, it will export `api.tasks.newtask` and `api.tasks.readtask`:

```typescript mesosphere/tasks.ts
import { readFunction, writeFunction } from "mesosphere/reactlibrary";
import { type } from "mesosphere/type";

// Add a task to the database with a task name and a boolean to indicate if it succeed or not.
export const newtask = writeFunction({
  args: {
    task: type.string(),
    succeed: type.boolean(),
  },
  handler: async (mesosphere, args) => {
    await mesosphere.database.add("tasks", {
      task: args.task,
      succeed: args.succeed,
    });
  },
});

// Read all tasks from the database.
export const readtask = readFunction({
  args: {},
  handler: async (database) => {
    return await database.get("tasks");
  },
});
```

**What it does:** It creates a new task in the database with a task name and a boolean to indicate if it succeed or not, and it exports a read function that returns all tasks from the database.

5. **Update the `app/routes/__root.tsx` file.** Add the `QueryClient` to the router to make React Query available anywhere in your TanStack Start app:

```typescript app/routes/__root.tsx
import { QueryClient } from "@tanstack/react-query";
import { createRootRouteWithContext } from "@tanstack/react-router";
import { Outlet, Scripts, HeadContent } from "@tanstack/react-router";
import * as React from "react";

export const Route = createRootRouteWithContext<{
  queryClient: QueryClient;
}>()({
  head: () => ({
    meta: [
      {
        charSet: "utf-8",
      },
      {
        name: "viewport",
        content: "width=device-width, initial-scale=1",
      },
      {
        title: "TanStack Start Starter",
      },
    ],
  }),
  component: RootComponent,
});

function RootComponent() {
  return (
    <RootDocument>
      <Outlet />
    </RootDocument>
  );
}

function RootDocument({ children }: { children: React.ReactNode }) {
  return (
    <html>
      <head>
        <HeadContent />
      </head>
      <body>
        {children}
        <Scripts />
      </body>
    </html>
  );
}
```

6. **Update the `app/router.tsx` file.** create a ReactClient and pass it to the MesosphereProvider and wrap your app with it:

```typescript app/router.tsx
import { createRouter } from "@tanstack/react-router";
import { QueryClient } from "@tanstack/react-query";
import { routerWithQueryClient } from "@tanstack/react-router-with-query";
import { MesosphereProvider, ReactClient } from "mesosphere/reactlibrary";
import { routeTree } from "./routeTree.gen";

export function getRouter() {
  const MESOSPHERE_URL = (import.meta as any).env.VITE_MESOSPHERE_URL!;
  const MESOSPHERE_API_KEY = (import.meta as any).env.VITE_MESOSPHERE_API_KEY!;

  if (!MESOSPHERE_URL) {
    console.error("missing envar VITE_MESOSPHERE_URL");
  }
  if (!MESOSPHERE_API_KEY) {
    console.error("missing envar VITE_MESOSPHERE_API_KEY");
  }

  const mesosphere = new ReactClient(MESOSPHERE_URL, MESOSPHERE_API_KEY);

  const queryClient: QueryClient = new QueryClient({
    defaultOptions: {
      queries: {
        queryKeyHashFn: mesosphere.hashFn(),
        queryFn: mesosphere.queryFn(),
      },
    },
  });
  mesosphere.connect(queryClient);

  const router = routerWithQueryClient(
    createRouter({
      routeTree,
      defaultPreload: "intent",
      context: { queryClient },
      scrollRestoration: true,
      Wrap: ({ children }) => (
        <MesosphereProvider client={mesosphere.mesosphereClient}>
          {children}
        </MesosphereProvider>
      ),
    }),
    queryClient,
  );

  return router;
}
```

7. **Use the functions in your TanStack Start app.** in the `app/routes/index.tsx` file, you can use the `newtask` and `readtask` functions to add and read tasks from the database:

```typescript app/routes/index.tsx
import { callread, callwrite } from "mesosphere/reactlibrary";
import { useSuspenseQuery } from "@tanstack/react-query";
import { useMutation } from "@tanstack/react-query";
import { createFileRoute } from "@tanstack/react-router";
import { api } from "../../mesosphere/deploy";

export const Route = createFileRoute("/")({
  component: Home,
});

function Home() {
  const { readtask } = useSuspenseQuery(callread(api.tasks.readtask, {}));
  const { newtasks, isPending } = useMutation({
    writeFunction: callwrite(api.tasks.newtask),

}
```
