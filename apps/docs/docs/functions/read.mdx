Read functions can fetch and get data from the database, It can perform complex logical operations such as checking the authentication status of the user accessing your application, and much more; it can also return data to the client.

Here is an example Read function that take one argument the id of the users, read the data from the database and return the result:

```typescript mesosphere/users.ts
import { readFunction } from "mesosphere/reactlibrarie";
import { type } from "mesosphere/type";

// Read the database and return 10 results, filtering by user ID.
export const newuser = readFunction({
  args: { usersListId: type.id("users") },
  handler: async (mesosphere, args) => {
    const users = await mesosphere.database
      .get("users")
      .filter("by_users_list_id", (fb) =>
        fb.content("usersListId", args.usersListId),
      )
      .order("desc")
      .nresult("10")
      .accumulate();
    return users;
  },
});
```

Try to understand this code; if you don't understand it, run it yourself.

# Read function

To make a read function in Mesosphere you use the `readFunction` function, you pass it an object with an `handler` that will return the result of your read function.

Here is an example:

```typescript mesosphere/yourfunction.ts
import { readFunction } from "mesosphere/reactlibrarie";

export const yourreadfunction = readFunction({
  args: {},
  handler: () => {
    return "Wow thats a really cool code";
  },
});
```

## Arguments

Read functions accept named arguments, the arguments are accessible as fields parameter in the `handler`.

Here is an example:

```typescript mesosphere/yourfunction.ts
import { readFunction } from "mesosphere/reactlibrarie";

export const yourreadfunction = readFunction({
  args: { a: number, b: number },
  handler: (args) => {
    return args.a + args.b;
  },
});
```

In the previous example your arguments are not validated, to validate them you use the `type` function.

Here is an example:

```typescript mesosphere/yourfunction.ts
import { readFunction } from "mesosphere/reactlibrarie";
import { type } from "mesosphere/type";

export const yourreadfunction = readFunction({
  args: { a: type.number(), b: type.number() },
  handler: (args) => {
    return args.a + args.b;
  },
});
```

See the [type function](./type) for the full list of supported type.

## Read context

The `readFunction` can read data from the database, it can be use in other part of Mesosphere by passing a `ReadMesosphere` object to the handler function as the first parameter.

Here is an example:

```typescript mesosphere/yourfunction.ts
import { readFunction } from "mesosphere/reactlibrarie";
import { type } from "mesosphere/type";

export const yourreadfunction = readFunction({
  args: { a: type.number(), b: type.number() },
  handler: (mesosphere, args) => {
    // Your read object using the arguments will go here
    // Use `mesosphere`
  },
});
```

To read and fetch data from the database use the `database` field, we change the handler to an async function so we can await the promise returned by `mesosphere.database.get()`.

Here is an example:

```typescript mesosphere/yourfunction.ts
import { readFunction } from "mesosphere/reactlibrarie";
import { type } from "mesosphere/type";

export const yourreadfunction = readFunction({
  args: { name: type.string() },
  handler: async (mesosphere, args) => {
    return await mesosphere.database
      .get("users", { name: args.name })
      .accumulate();
  },
});
```

# Call your read function from your clients

To call your read function form a [react](../client/react/react) client use the `callread` hook and the generated `api` object from `deploy`.

Here is an example:

```typescript src/App.tsx
import { callread } from "mesosphere/reactlibrarie";
import { api } from "../mesosphere/deploy";

export default function App() {
  const readdata = callread(api.yourfunction.yourreadfunction);
  const handleClick = () => {
    readdata({ name: Theo });
  };
  return (
    <div className="App">
      <button
        onClick={() => handleClick()}>Read user
      </button>;
    </div>
  );
}
```

Try this example and see what happens.

See the full [React](../client/react/react) client documentation for more information.

# Caching and Realtime

Read functions have two main attributes:

1. **Caching:** Mesosphere automatically caches the result of read function. If your client call the same read function multiple time with the same arguments it will receive the caches result

2. **Realtime:** All Mesosphere clients can `hug` the data feed to always stay up to date with the new data in the database.

For all of that to work, the read function is deterministic. That means the `handler` will always return the same result given the same arguments.

It can't call third party services, if you want to use third party services use [side actions](./sideactions).
