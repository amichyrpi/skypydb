---
title: "Type validation"
description: "Validate Argument and Return Type"
---

Argument and return type validation ensures that every database operation can be called with the correct arguments and returns the same types of values.

Type validation is important; without argument type validation, a malicious user can call your public functions with unexpected arguments and can cause data leakage. We provide type validation because at runtime (on our server) TypeScript's type system is not present. For a secure app, we recommend adding type validation to your arguments in all of your public functions; for [close functions,](./closefunctions) you can use type validation if you want, but it's not needed.

# Supported types

Both public and close functions accept and return the following types, the following types can be accessed with the `type` object from `"mesosphere/type"`.

Here is a table listing all the supported types:

| Mesosphere Type | Example Usage          | Types validator                            | Json format     | Tips and tricks                                                                                                            |
| --------------- | ---------------------- | ------------------------------------------ | --------------- | -------------------------------------------------------------------------------------------------------------------------- |
| Id              | `doc._id`              | `type.id("tableName")`                     | string          | Unique identifier referencing a row in a table.                                                                            |
| Null            | `null`                 | `type.null()`                              | null            | Represents the absence of a value. `undefined` is not a valid Mesosphere value.                                            |
| Int64           | `3n`                   | `type.int64()`                             | string (base10) | Supports signed 64-bit integers between −2^63 and 2^63−1. Stored as strings when serialized to JSON to preserve precision. |
| Float64         | `1.2`                  | `type.number()`                            | number / string | IEEE-754 double-precision floating point. Special values like `NaN` or `Infinity` may be serialized as strings.            |
| Boolean         | `true`                 | `type.boolean()`                           | bool            | Boolean values can be `true` or `false`.                                                                                   |
| String          | `"som"`                | `type.string()`                            | string          | UTF-8 encoded strings. Must be valid Unicode sequences.                                                                    |
| Bytes           | `new ArrayBuffer(8)`   | `type.bytes()`                             | string (base64) | Binary data encoded as base64 when serialized to JSON.                                                                     |
| Array           | `[1, 2.3, "som"]`      | `type.array(values)`                       | array           | Arrays can contain up to **8192 values**.                                                                                  |
| Object          | `{a: "som"}`           | `type.object({property: value})`           | object          | Structured object with predefined properties.                                                                              |
| Record          | `{"a": "1", "b": "2"}` | `type.record(keys, values)`                | object          | Dynamic object with typed keys and values.                                                                                 |
| Any             | `"som"` / `1` / `{}`   | `type.any()`                               | any JSON value  | Accepts any valid Mesosphere value. Use sparingly since it disables type validation.                                       |
| Union           | `"som"` / `42`         | `type.union(type.string(), type.number())` | value           | Value must match **one of the provided validators**.                                                                       |
| Literal         | `"som"`                | `type.literal("som")`                      | value           | Accepts only the exact literal value specified.                                                                            |
| Optional        | `undefined` / `"som"`  | `type.optional(type.string())`             | value / missing | Field may be omitted from objects or contain a valid value.                                                                |

# Use type validation

To enable type validation for function arguments, provide an object with `args` and `handler` properties to the `write`, `read`, or `upload` function. To return type validation, use the `returns` property in the object.

Here is a simple example that use type validation for both argument and return values:

```typescript mesosphere/users.ts
import { writeFunction } from "mesosphere/reactlibrarie";
import { type } from "mesosphere/type";

export const newuser = writeFunction({
  args: {
    name: type.string(),
    email: type.string(),
  },
  returns: v.null(),
  handler: async (mesosphere, args) => {
    await mesosphere.database.add("users", {
      name: args.name,
      email: args.email,
    });
  },
});
```

This example shows that you don't need TypeScript type annotations.

Mesosphere handles type validation for you.

# Other examples

**Any**

Any is a field that can take on any type of value, it can be use with `type.any()`.

Here is an example:

```typescript mesosphere/examplefunction.ts
import { writeFunction } from "mesosphere/reactlibrarie";
import { type } from "mesosphere/type";

export const yourfunction = writeFunction({
  args: {
    yourvalue: type.any(),
  },
  handler: async (mesosphere, args) => {
    // ... Do something with your value
  },
});
```

**Optional**

Optional is an optional field that can contain any other type; it can be used with `type.optional(...)`.

Here is an example:

```typescript mesosphere/examplefunction.ts
import { writeFunction } from "mesosphere/reactlibrarie";
import { type } from "mesosphere/type";

export const yourfunction = writeFunction({
  args: {
    optionalstringvalue: type.optional(type.string()),
    optionalnumbervalue: type.optional(type.number()),
  },
  handler: async (mesosphere, args) => {
    // ... Do something with your optional values
  },
});
```

**Unions**

Unions is a field that can contain multiple other type; it can be used with `type.union()`.

Here is an example:

```typescript mesosphere/examplefunction.ts
import { writeFunction } from "mesosphere/reactlibrarie";
import { type } from "mesosphere/type";

export const yourfunction = writeFunction({
  args: {
    multiplevalues: type.union(type.string(), type.number(), type.null()),
  },
  handler: async (mesosphere, args) => {
    // ... Do something with your values
  },
});
```

**Literal**

Literal is a field that is a constant; it can be used with `type.literal()`. Literal values are best used in combination with `type.union`.

Here is an example:

```typescript mesosphere/examplefunction.ts
import { writeFunction } from "mesosphere/reactlibrarie";
import { type } from "mesosphere/type";

export const yourfunction = writeFunction({
  args: {
    multiplevalues: type.union(
      type.literal("som1"),
      type.literal("som2"),
      type.literal("som3"),
    ),
  },
  handler: async (mesosphere, args) => {
    // ... Do something with your literal values
  },
});
```

**Record**

Record is a field that is can assign arbitrary keys to values; it can be used with `type.record()`.

Here is an example:

```typescript mesosphere/examplefunction.ts
import { writeFunction } from "mesosphere/reactlibrarie";
import { type } from "mesosphere/type";

export const yourfunction = writeFunction({
  args: {
    assignvalue: type.record(type.string(), type.boolean()),
  },
  handler: async (mesosphere, args) => {
    // ... Do something with your record value
  },
});
```
