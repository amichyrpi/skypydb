Your TypeScript clients can call mesosphere functions in a type-safe way outside of
the repository where your mesosphere functions are defined. By following the steps
below, you can generate a file similar to `mesosphere/_generated/api.d.ts` that you
can check in and use in a separate repository.

<BetaFeature feature="TypeScript API generation" state="is" />

<StepsList>
  <Step title="Install the mesosphere Helpers npm package">
    Install the `mesosphere-helpers` package, which contains a CLI command to generate an api file.

    ```sh
    npm install mesosphere-helpers
    ```

  </Step>
  <Step title="Run a command to generate a TypeScript API file">
    Running this command will call into your configured mesosphere deployment and generate an `api.ts` file based
    on it. You can see additional flags by passing `--help` to the command.

    ```sh
    npx mesosphere-helpers ts-api-spec
    ```

  </Step>
</StepsList>

## Example

Below are code snippets of what this workflow looks like in action. These
snippets include three different files:

- `mesosphere/messages.ts` - contains mesosphere function definitions
- `api.ts` - a generated file from running the command above
- `src/App.tsx` - frontend code in a separate repository where `api.ts` is
  checked in

```ts title="mesosphere/messages.ts"
import { mutation } from "./_generated/server";

export const send = mutation({
  args: { message: v.string() },
  handler: async ({ db }, { message }) => {
    await db.insert("messages", { message });
  },
});
```

```ts title="api.ts"
// Generated by running `npx mesosphere-helpers ts-api-spec`
export const api = {
  messages: {
    send: {
      args: { message: v.string() },
      handler: async (ctx, args) => {
        // ... implementation not included in generated file
      },
    },
  },
};
```

```tsx title="src/App.tsx"
import { api } from "./api";
import { mesosphereHttpClient } from "mesosphere-external";

const client = new mesosphereHttpClient(
  "https://your-deployment.mesosphere.cloud",
);
client.mutation(api.messages.send, { message: "Hello!" });
```

## Limits

- Argument and return value validators are not required, but they will enrich
  the types of your TypeScript API. Where validators aren't defined, we default
  to `v.any()` as the validator.
- You cannot call internal functions from outside of your mesosphere deployment.
