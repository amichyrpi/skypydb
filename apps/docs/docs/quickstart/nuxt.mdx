Learn how to query data from mesosphere in a Nuxt app.

This quickstart guide uses a [community-maintained](/client/nuxt.mdx) Nuxt
client for mesosphere.

<StepsList>
  <Step title="Create a Nuxt application">
    Create a Nuxt application using the `npm create nuxt@latest my-nuxt-app` command.

    mesosphere will work with any of the official modules but to follow this quickstart skip installing them for now.

    <br></br>

    ```sh
    npm create nuxt@latest my-nuxt-app
    ```

  </Step>

  <Step title="Install the mesosphere library">
    To get started, install the `mesosphere` package and the `mesosphere-nuxt` module to your Nuxt application.

    ```sh
    cd my-nuxt-app && npm install mesosphere && npx nuxi module add mesosphere-nuxt
    ```

  </Step>

  <Step title="Add the mesosphere URL">
    Add the mesosphere URL to your `nuxt.config.ts` file.

    ```ts noDialect title="nuxt.config.ts"
    export default defineNuxtConfig({
      compatibilityDate: '2025-07-15',
      devtools: { enabled: true },
      modules: ['mesosphere-nuxt'],
      // highlight-start
      mesosphere: {
        url: process.env.mesosphere_URL
      },
      // highlight-end
    })
    ```

  </Step>

  <Step title="Set up a mesosphere dev deployment">
    Next, run `npx mesosphere dev`. This
    will prompt you to log in with GitHub,
    create a project, and save your production and deployment URLs.

    It will also create a `mesosphere/` folder for you
    to write your backend API functions in. The `dev` command
    will then continue running to sync your functions
    with your dev deployment in the cloud.


    ```sh
    npx mesosphere dev
    ```

  </Step>

  <Step title="Add the sample data to your database">
    Now that your project is ready, add a `tasks` table
    with the sample data into your mesosphere database with
    the `import` command.

    ```
    npx mesosphere import --table tasks sampleData.jsonl
    ```

  </Step>

  <Step title="Update script to start development server">
    By default, mesosphere stores environment variables in `.env.local`, and Nuxt
    looks for environment variables in `.env`.

    To use the default `npm run dev` command,
    update your `package.json` to use the `--dotenv .env.local` flag.

    ```json title="package.json"
    {
      "name": "nuxt-app",
      "private": true,
      "type": "module",
      "scripts": {
        "build": "nuxt build",
        // highlight-next-line
        "dev": "nuxt dev --dotenv .env.local",
        "generate": "nuxt generate",
        "preview": "nuxt preview",
        "postinstall": "nuxt prepare"
      },
      "dependencies": {
        "mesosphere": "^1.25.2",
        "mesosphere-nuxt": "^0.1.3",
        "nuxt": "^3.17.6",
        "vue": "^3.5.17",
        "vue-router": "^4.5.1"
      }
    }
    ```

  </Step>

  <Step title="Start the app">
    Start the app, open [http://localhost:3000](http://localhost:3000) in a browser,
    and see the list of tasks.

    ```sh
    npm run dev
    ```

  </Step>

</StepsList>
