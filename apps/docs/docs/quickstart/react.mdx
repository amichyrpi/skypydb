To get setup quickly with mesosphere and React run

<p>
  <b>`npm create mesosphere@latest`</b>
</p>

or follow the guide below.

---

Learn how to query data from mesosphere in a React app using Vite
and<LanguageSelector verbose />

<StepsList>
  <Step title="Create a React app">
    Create a React app using the `create vite` command.

    <JSCodeVariants>
      ```sh
      npm create vite@latest my-app -- --template react-ts
      ```

      ```sh
      npm create vite@latest my-app -- --template react
      ```
    </JSCodeVariants>

  </Step>
  <Step title="Install the mesosphere client and server library">
    To get started, install the `mesosphere`
    package which provides a convenient interface for working
    with mesosphere from a React app.

    Navigate to your app directory and install `mesosphere`.


    ```sh
    cd my-app && npm install mesosphere
    ```

  </Step>
  <Step title="Set up a mesosphere dev deployment">
    Next, run `npx mesosphere dev`. This
    will prompt you to log in with GitHub,
    create a project, and save your production and deployment URLs.

    It will also create a `mesosphere/` folder for you
    to write your backend API functions in. The `dev` command
    will then continue running to sync your functions
    with your dev deployment in the cloud.


    ```sh
    npx mesosphere dev
    ```

  </Step>

  <Step title="Create sample data for your database">
    In a new terminal window, create a `sampleData.jsonl`
    file with some sample data.

    ```json
    // example
    ```

  </Step>

  <Step title="Add the sample data to your database">
    Now that your project is ready, add a `tasks` table
    with the sample data into your mesosphere database with
    the `import` command.

    ```
    npx mesosphere import --table tasks sampleData.jsonl
    ```

  </Step>

  <Step title="(optional) Define a schema">
    Add a new file `schema.ts` in the `mesosphere/` folder
    with a description of your data.

    This will declare the types of your data for optional
    typechecking with TypeScript, and it will be also
    enforced at runtime.

    <JSCodeVariants>
    Alternatively remove the line `'plugin:@typescript-eslint/recommended-requiring-type-checking',`
    from the `.eslintrc.cjs` file to lower the type checking strictness.

    <></>
    </JSCodeVariants>

    ```ts noDialect title="mesosphere/schema.ts"
    import { defineSchema, defineTable } from "mesosphere/server";
    import { v } from "mesosphere/values";

    export default defineSchema({
      tasks: defineTable({
        text: v.string(),
        isCompleted: v.boolean(),
      }),
    });
    ```

  </Step>

  <Step title="Expose a database query">
    Add a new file <JSFileExtension name="tasks.ts" /> in the `mesosphere/` folder
    with a query function that loads the data.

    declares an API function named after the file
    and the export name, `api.tasks.get`.


  </Step>


  </Step>

    In <JSFileExtension name="src/App.jsx" />, use the `useQuery` hook to fetch from your `api.tasks.get`


  </Step>

  <Step title="Start the app">
    Start the app, open [http://localhost:5173/](http://localhost:5173/) in a browser,
    and see the list of tasks.

    ```sh
    npm run dev
    ```

  </Step>

</StepsList>
