Learn how to query data from mesosphere in a Svelte app.

<StepsList>
  <Step title="Create a SvelteKit app">
    Create a SvelteKit app using the `npx sv create` command.

    Other sets of options will work with the library but for this quickstart guide:

    - For "Which Svelte app template," choose **"SvelteKit minimal."**
    - For a package manager, choose **"npm."**
    - For "Add type checking with TypeScript," choose **"Yes, using TypeScript syntax."**
    - For "Select additional options," you don't need to enable anything.

    <br></br>

    ```sh
    npx sv@latest create my-app
    ```

  </Step>

  <Step title="Install the mesosphere client and server library">
    To get started, install the `mesosphere` and `mesosphere-svelte` packages.

    ```sh
    cd my-app && npm install mesosphere mesosphere-svelte
    ```

  </Step>

  <Step title="Customize the mesosphere path">
    SvelteKit doesn't like referencing code outside of source, so customize
    the mesosphere functionsDir to be under `src/`.

    ```json
    // example
    ```

  </Step>

  <Step title="Set up a mesosphere dev deployment">
    Next, run `npx mesosphere dev`. This
    will prompt you to log in with GitHub,
    create a project, and save your production and deployment URLs.

    It will also create a `src/mesosphere/` folder for you
    to write your backend API functions in. The `dev` command
    will then continue running to sync your functions
    with your dev deployment in the cloud.


    ```sh
    npx mesosphere dev
    ```

  </Step>

  <Step title="Create sample data for your database">
    In a new terminal window, create a `sampleData.jsonl`
    file with some sample data.


  </Step>

  <Step title="Add the sample data to your database">
    Now that your project is ready, add a `tasks` table
    with the sample data into your mesosphere database with
    the `import` command.

    ```
    npx mesosphere import --table tasks sampleData.jsonl
    ```

  </Step>

  <Step title="Start the app">
    Start the app, open [http://localhost:5173](http://localhost:5173) in a browser,
    and see the list of tasks.

    ```sh
    npm run dev
    ```

  </Step>
