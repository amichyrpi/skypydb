title: "Scheduler"

In Mesosphere, you can schedule your functions to run in the future. This allows you to build long-running workflows without the need to set up or maintain queues or other infrastructure.

Scheduled functions are stored directly in the database, which means they are not prone to failure due to unexpected downtime or system crashes.

# Scheduled Functions

You can schedule public or closed functions by using `writeFunction` or `closewrite`, and the `scheduler` object inside the `handler`.

When scheduling a function, you can use:

- `rundelay`, which schedules your function to run after a specified delay. The delay is measured in milliseconds.

- `runprecise`, which schedules your function to run at a specific date or timestamp. The time is measured in milliseconds.

You pass the path of the function to the `scheduler` like this:

`mesosphere.scheduler.rundelay(1000, closefunction.message.updateTemporaryMessage, { messageId: id })`

By the way, you can schedule up to **5,000 functions** with a single scheduler — pretty crazy, right?

Here is an example:

```typescript mesosphere/message.ts
import { writeFunction, closewrite } from "mesosphere/reactlibrary";
import { closefunction } from "./mesosphere/deploy";
import { type } from "mesosphere/type";

export const newTemporaryMessage = writeFunction({
  args: {
    user: type.string(),
    body: type.string(),
  },
  handler: async (mesosphere, { body, author }) => {
    const id = await mesosphere.database.add("messages", {
      user,
      body,
    });
    await mesosphere.scheduler.rundelay(
      1000,
      closefunction.messages.updateTemporaryMessage,
      {
        messageId: id,
      },
    );
  },
});

export const updateTemporaryMessage = closewrite({
  args: {
    messageId: type.id("messages"),
  },
  handler: async (mesosphere, args) => {
    await mesosphere.database.delete("messages", args.messageId);
  },
});
```

Here is an example of how to use `runprecise`.

```typescript mesosphere/message.ts
import { writeFunction, closewrite } from "mesosphere/reactlibrary";
import { closefunction } from "./mesosphere/deploy";
import { type } from "mesosphere/type";

export const newTemporaryMessage = writeFunction({
  args: {
    user: type.string(),
    body: type.string(),
  },
  handler: async (mesosphere, { body, author }) => {
    const id = await mesosphere.database.add("messages", {
      user,
      body,
    });
    await mesosphere.scheduler.runprecise(
      new Date("2090-17-03T00:00:00Z"),
      // The end of the world will happen at this time… just kidding.
      closefunction.messages.updateTemporaryMessage,
      {
        messageId: id,
      },
    );
  },
});

export const updateTemporaryMessage = closewrite({
  args: {
    messageId: type.id("messages"),
  },
  handler: async (mesosphere, args) => {
    await mesosphere.database.delete("messages", args.messageId);
  },
});
```

# Scheduled Security

Scheduled functions provide failure safety. If your write operation fails, the scheduler will not schedule the function. On the other hand, if your write operation succeeds, the scheduler will reliably schedule your function.

However, unlike write operations, side actions can access third-party services. Because of this, even if a side action fails, the scheduler will still schedule the function.

Be careful when using scheduled functions. If used incorrectly, they can break your workflows. That’s why we provide this guide.

# Access Your Scheduled Metadata

When a scheduled a function in Mesosphere, it is stored as a document in the `_scheduler` system table.

You can access the scheduled function metadata using a [read function](../functions/read) by calling `mesosphere.database.st.get()`.

The `st` namespace stands for [system table](../database/systemtable).

Here is an example:

```typescript mesosphere/message.ts
import { readFunction } from "mesosphere/reactlibrary";
import { type } from "mesosphere/type";

export const readscheduledmetadata = readFunction({
  args: {},
  handler: async (mesosphere, args) => {
    return await mesosphere.database.st.get("_scheduler").accumulate();
  },
});
```

## Returned document

```json
{
  "_id": "1044c087-ed64-4592-8d48-3b3513186e80",
  "_creationTime": 1773178195,
  "name": "mesosphere:messages.ts:newTemporaryMessage",
  "scheduledTime": 1699931054657,
  "state": "success",
  "completedTime": 1699931054690.366
}
```

Let's break this down:

This output contains:

- `_id`: a randomly generated UUID.

- `_creationTime`: the time when the scheduled function was created.

- `name`: the name of the scheduled function and the file where the function is defined.

- `scheduledTime`: the timestamp indicating when the function is scheduled to run.

- `completedTime`: the timestamp indicating when the scheduled function finished successfully.

- `state`: the current state of the scheduled function. It can be:
  - `"success"`: the function ran successfully.
  - `"pending"`: the function is waiting for its scheduled start time.
  - `"executing"`: the function is currently being executed by the server.
  - `"failed"`: the function failed to start or execute.

**Important**

All metadata for scheduled functions that have been executed is removed from the cache 24 hours after they run. Their state at that point no longer matters.

However, their logs are stored directly in the [Mesosphere dashboard](https://mesosphere.ahen-studio.com/dashboard).
