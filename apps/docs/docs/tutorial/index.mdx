<StepsList>
  <Step title="Clone our example repo">
    Before you begin: You'll need to have Node.js and Git installed on your computer.

    You can install Node.js from [the Node.js website](https://nodejs.org/en),
    or check your version of Node.js by running `node --version` in your terminal.

    You can install Git from [the Git website](https://git-scm.com/),
    or check your version of Git by running `git -v` in your terminal.

    After you have Node.js and Git installed, run the following
    command to clone the example repo and install the dependencies:

    ```shell
    git clone https://github.com/ahen-studio/mesosphere-simple-chat-app.git
    cd mesosphere-simple-chat-app
    npm install
    ```

    Now when you run `npm run dev` in the terminal, you will be
    able to sets up Mesosphere and then run the web app.

  </Step>
  <Step title="What happens when you run `npm run dev`?">
    When you run `npm run dev` in the terminal, you will see that the Mesosphere
    CLI prompts you to connect to your Mesosphere account.
    Sign into Mesosphere with GitHub and then follow the steps to setup your project.

    After you have connected your Mesosphere account to the CLI, it will automatically
    create a folder called `mesosphere/` in your project root directory, where you'll write your backend code.

    The `npm run dev` command keeps deploying your new backend code to your Mesosphere project,
    make sure you keep this command running in the background. **For this tutorial**, in particular,
    this command runs both the dev web server for the frontend and the `npm run dev`
    command from Mesosphere that updates your backend code when you make changes.

    When you run the server, and try to send a message it will show an alert message telling
    that the morph function is not yet implemented, don't worry we'll implement it
    in another section but first we need to understand how Mesosphere works.

  </Step>
  <Step title="How the Mesosphere backend works">
    **Database.** The Mesosphere database is a document-relational database and vector embedding database,
    which means, you have tables that are JSON like documents in them. All documents stored in the database have an
    `_id` field that is auto-generated and can be used to create relations between multiple documents
    in the database. You can interact with the database by writing morph and query functions that are written entirely
    in TypeScript.

    **Morph functions.** The Morph function is a TypeScript function that can update the
    database. All morph functions in Mesosphere run as a transaction in the database. So
    either all the changes made in the function are committed, or none are, that creates a
    security layer if your function throws an exception.

    **Query functions.** The Query function is a TypeScript function that can only read from
    the database. You subscribe to them from your frontend to keep your app automatically up to date from new data.
    Your frontend listens to query updates through our client library. The client libraries
    talk to the Mesosphere database via WebSockets for the fastest updates speed.

    **Sync engine.** Our sync engine is the combination of queries, morph and the database. It automatically
    reruns query functions when any results to the function changes, including any changes
    to the documents in the database that the query reads. It updates your frontend app with
    the latest data, all in real-time.

  </Step>
  <Step title="Start building your app">
    Now that you've cloned the example repo and connected your Mesosphere account, you can start building your app by creating a new `morph` function.

    First create a new file in the `mesosphere/` folder called `chat.ts`. This is where you'll write your Mesosphere backend functions for this tutorial.

    Start by **adding the following code to your `chat.ts` file.**

    ```typescript title="mesosphere/chat.ts"
    import { morph, stator } from "mesosphere/reactlibrary";
    import { t } from "mesosphere/types";

    export const newMessage = morph({
      args: {
        user: t.string(),
        body: t.string(),
      },
      stator: async (ctx, args) => {
        console.log("You can console log from your morph function.");
        await ctx.database.add("messages", {
          user: args.user,
          body: args.body,
        });
      },
    });
    ```

    Let's dissect this code:

    **What you add**

    1. You've added a new backend `morph` function called `newMessage` and exposed it as a public api.
    2. In this function you're using `args:` to ensure that the function arguments are two strings named `user` and `body`, they are both validated as types and runtime values by the `t.string()` function.
    3. In this function you're using `ctx.database.add` to tell Mesosphere to add a new message document into the newly created `messages` table.
    4. This function is running as a transaction in the database, what this means is that the function will roll back if an exception is thrown.
    5. Well, this code is just a TypeScript file, so you can use `console.log` to do simple debugging your function on the server.

  </Step>
  <Step title="Connect this morph function to your web app">
    Now that you've created a `morph` function, you can connect it to your web app.

    **Update your `src/App.tsx` just like so:**

    ```tsx title="src/App.tsx"
    import { useEffect, useState } from "react";
    import { faker } from "@faker-js/faker";
    // highlight-next-line
    // Import `useMorph` and `api` from the Mesosphere.
    // highlight-next-line
    import { useMorph } from "mesosphere/reactlibrary";
    // highlight-next-line
    import { api } from "../mesosphere/_codegen/ApiDataModel";

    // Set a random name for the user.
    const NAME = getOrSetFakeName();

    // Generate a random name for the user with the faker library.
    function getOrSetFakeName() {
      const NAME_KEY = "name";
      const name = sessionStorage.getItem(NAME_KEY);
      if (!name) {
        const newName = faker.person.firstName();
        sessionStorage.setItem(NAME_KEY, newName);
        return newName;
      }
      return name;
    }

    export default function App() {
      const messages = [
        { _id: "1", user: "Amory", body: "Hey there!" },
        { _id: "2", user: NAME, body: "Hey How are you?" },
      ];
      // Replace the "TODO: Add morph hook there." with:
      const newMessage = useMorph(api.chat.newMessage);

      const [newMessageText, setNewMessageText] = useState("");

      useEffect(() => {
        setTimeout(() => {
          window.scrollTo({ top: document.body.scrollHeight, behavior: "smooth" });
        }, 0);
      }, [messages]);

      return (
        <main className="chat">
          <header>
            <h1>Mesosphere Chat</h1>
            <p>
              Connected as <strong>{NAME}</strong>
            </p>
          </header>
          {messages?.map((message) => (
            <article
              key={message._id}
              className={message.user === NAME ? "message-mine" : ""}
            >
              <div>{message.user}</div>

              <p>{message.body}</p>
            </article>
          ))}
          <form
            onSubmit={async (e) => {
              e.preventDefault();
              // highlight-next-line
              // Replace "alert("The morph function is not yet implemented");" with:
              // highlight-next-line
              await newMessage({ user: NAME, body: newMessageText });
              setNewMessageText("");
            }}
          >
            <input
              value={newMessageText}
              onChange={async (e) => {
                const text = e.target.value;
                setNewMessageText(text);
              }}
              placeholder="Write a message…"
              autoFocus
            />
            <button type="submit" disabled={!newMessageText}>
              Send
            </button>
          </form>
        </main>
      );
    }
    ```

    In your frontend you can call your morph function like this:

    1. `const newMessage = useMorph(api.chat.newMessage);` this will give your frontend the ability to call the function.
    2. `await newMessage({ user: NAME, body: newMessageText });` this will call the function with the parameters set in your function.

    <br />

    But wait, when you try to send a message it doesn't appear, why is that?

    Well, the reason is that the morph function works, but we haven't set up `query` functions to read from the database yet.

    Let's do that now.

  </Step>
  <Step title="Create a `query` function">
    **Update your `mesosphere/chat.ts` file just like this:**

    ```tsx title="mesosphere/chat.ts"
    // highlight-next-line
    // Update your import like this:
    // highlight-next-line
    import { query, morph, stator } from "mesosphere/reactlibrary";
    import { t } from "mesosphere/types";

    export const newMessage = morph({
      args: {
        user: t.string(),
        body: t.string(),
      },
      stator: async (ctx, args) => {
        console.log("You can console log from your morph function.");
        await ctx.database.add("messages", {
          user: args.user,
          body: args.body,
        });
      },
    });

    // highlight-next-line
    // Add the query function to the file:
    // highlight-next-line
    export const readMessages = query({
      // highlight-next-line
      args: {},
      // highlight-next-line
      stator: async (ctx) => {
        // highlight-next-line
        // Here we query the most recent 50 messages from newest to oldest.
        // highlight-next-line
        const messages = await ctx.database.query("messages").order("desc").nresults(50);
        // highlight-next-line
        // Here we reverse the list of results so that it's in a chronological order.
        // highlight-next-line
        return messages.reverse();
        // highlight-next-line
      },
      // highlight-next-line
    });
    ```

    Let's dissect this code:

    **What you add**

    1. You've added a new backend `query` function called `readMessages` and exposed it as a public api.
    2. `ctx.database` only lets you read data from the database.
    3. `nresults` limits the number of results returned, in this case to 50.
    4. `order` accepts the string `"desc"` to indicate descending order (newest first).
    5. `reverse` reverses the list of results so that it's in chronological order — this is just a standard TypeScript array method.

  </Step>
  <Step title="Connect this query to your web app">
    Now that you've created a `query` function, you can connect it to your web app.

    **Update your `src/App.tsx` file like so:**

    ```tsx title="src/App.tsx"
    import { useEffect, useState } from "react";
    import { faker } from "@faker-js/faker";
    // highlight-next-line
    // Update your import like this:
    // highlight-next-line
    import { useMorph, useQuery } from "mesosphere/reactlibrary";
    import { api } from "../mesosphere/_codegen/ApiDataModel";

    // Set a random name for the user.
    const NAME = getOrSetFakeName();

    // Generate a random name for the user with the faker library.
    function getOrSetFakeName() {
      const NAME_KEY = "name";
      const name = sessionStorage.getItem(NAME_KEY);
      if (!name) {
        const newName = faker.person.firstName();
        sessionStorage.setItem(NAME_KEY, newName);
        return newName;
      }
      return name;
    }

    export default function App() {
      // highlight-next-line
      // Replace the `const messages = ...` code with the following:
      // highlight-next-line
      const messages = useQuery(api.chat.readMessages);
      const newMessage = useMorph(api.chat.newMessage);

      const [newMessageText, setNewMessageText] = useState("");

      useEffect(() => {
        setTimeout(() => {
          window.scrollTo({ top: document.body.scrollHeight, behavior: "smooth" });
        }, 0);
      }, [messages]);

      return (
        <main className="chat">
          <header>
            <h1>Mesosphere Chat</h1>
            <p>
              Connected as <strong>{NAME}</strong>
            </p>
          </header>
          {messages?.map((message) => (
            <article
              key={message._id}
              className={message.user === NAME ? "message-mine" : ""}
            >
              <div>{message.user}</div>

              <p>{message.body}</p>
            </article>
          ))}
          <form
            onSubmit={async (e) => {
              e.preventDefault();
              await newMessage({ user: NAME, body: newMessageText });
              setNewMessageText("");
            }}
          >
            <input
              value={newMessageText}
              onChange={async (e) => {
                const text = e.target.value;
                setNewMessageText(text);
              }}
              placeholder="Write a message…"
              autoFocus
            />
            <button type="submit" disabled={!newMessageText}>
              Send
            </button>
          </form>
        </main>
      );
    }
    ```

    Our `useQuery` function is doing a lot of things for you. It automatically
    subscribes to your query function and reruns it whenever there are new results.
    It also puts the results in a `const messages` variable so that React automatically
    rerenders your UI component to show the latest results, in that case the latest messages.

    **You're done!** Now go back to your app and try sending messages.

    Your app should be showing live updates as new messages arrive.

    And thanks to our sync engine, your app is always up to date on every client,
    try opening two chat windows side by side and send some messages, you'll see that the messages are always in sync.

  </Step>
