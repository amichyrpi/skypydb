  <Step title="What are indexed queries?">
    An index is a way to tell the database to create a lookup structure to
    filter data faster than if we were not using indexes. Let's say that in
    our chat app we wanted to build a way to look up the `messages` table from just one user,
    we will tell Mesosphere to index the `user` field in the `messages` table
    and write a query with the `useIndex` syntax to filter the data.

    ```tsx title="src/App.tsx"
    import { useEffect, useState } from "react";
    import { faker } from "@faker-js/faker";
    import { useMorph, useQuery } from "mesosphere/reactlibrary";
    import { api } from "../mesosphere/_codegen/ApiDataModel";

    // Set a random name for the user.
    const NAME = getOrSetFakeName();

    // Generate a random name for the user with the faker library.
    function getOrSetFakeName() {
      const NAME_KEY = "name";
      const name = sessionStorage.getItem(NAME_KEY);
      if (!name) {
        const newName = faker.person.firstName();
        sessionStorage.setItem(NAME_KEY, newName);
        return newName;
      }
      return name;
    }

    export default function App() {
      // highlight-next-line
      const [nameFilter, setNameFilter] = useState("");

      // highlight-next-line
      const messages = useQuery(api.chat.readMessages, { nameFilter });
      const newMessage = useMorph(api.chat.newMessage);

      const [newMessageText, setNewMessageText] = useState("");

      useEffect(() => {
        setTimeout(() => {
          window.scrollTo({ top: document.body.scrollHeight, behavior: "smooth" });
        }, 0);
      }, [messages]);

      return (
        <main className="chat">
          <header>
            <h1>Mesosphere Chat</h1>
            <p>
              Connected as <strong>{NAME}</strong>
            </p>
            // highlight-next-line
            <div className="user-filter">
              // highlight-next-line
              <input
              // highlight-next-line
                type="text"
                // highlight-next-line
                value={nameFilter}
                // highlight-next-line
                onChange={(e) => setNameFilter(e.target.value)}
                // highlight-next-line
                placeholder="Filter your messages by user"
                // highlight-next-line
                className="user-filter-input"
                // highlight-next-line
              />
              // highlight-next-line
            </div>
          </header>
          {messages?.map((message) => (
            <article
              key={message._id}
              className={message.user === NAME ? "message-mine" : ""}
            >
              <div>{message.user}</div>

              <p>{message.body}</p>
            </article>
          ))}
          <form
            onSubmit={async (e) => {
              e.preventDefault();
              await newMessage({ user: NAME, body: newMessageText });
              setNewMessageText("");
            }}
          >
            <input
              value={newMessageText}
              onChange={async (e) => {
                const text = e.target.value;
                setNewMessageText(text);
              }}
              placeholder="Write a message…"
              autoFocus
            />
            <button type="submit" disabled={!newMessageText}>
              Send
            </button>
          </form>
        </main>
      );
    }
    ```

    Here we modified the frontend to add a user filter input field. When the user
    types in the input field, the query is automatically updated to filter the
    messages by the user.

    But wait, when you input a user name in the input field, nothing happens.
    That's because we need to update the query function to use the new `nameFilter` arguments.

    ```typescript title="mesosphere/chat.ts"
    import { query, morph } from "mesosphere/reactlibrary";
    import { t } from "mesosphere/types";

    export const newMessage = morph({
      args: {
        user: t.string(),
        body: t.string(),
      },
      stator: async (ctx, args) => {
        console.log("You can console log from your morph function.");
        await ctx.database.add("messages", {
          user: args.user,
          body: args.body,
        });
      },
    });

    export const readMessages = query({
      // highlight-next-line
      args: { nameFilter: t.string() },
      // highlight-next-line
      stator: async (ctx, args) => {
        // highlight-next-line
        const messages = args.nameFilter
        // highlight-next-line
          ? await ctx.database
          // highlight-next-line
              .query("messages")
              // highlight-next-line
              .useIndex("by_user", (q) => q.eq("user", args.nameFilter))
              // highlight-next-line
              .order("desc")
              // highlight-next-line
              .nresults(50)
              // highlight-next-line
          : await ctx.database.query("messages").order("desc").nresults(50);
        // Here we reverse the list of results so that it's in a chronological order.
        return messages.reverse();
      },
    });
    ```

    Now that we've updated the query function, we need to add the index to
    the table, to do that we will use a database [schema](/database/schemas).

    To add the index to the table, we'll first create a new file called `schema.ts`
    in the `mesosphere/` folder, and then add the following code to it:

    ```typescript title="mesosphere/schema.ts"
    import { schema, table } from "mesosphere/serverside";
    import { t } from "mesosphere/types";

    export default schema({
      messages: table({
        user: t.string(),
        body: t.string(),
      }).index("by_user", ["user"]),
    });
    ```

    Now that we've created the schema and updated the query function, we can
    now test our app by putting the name of a user in the input field and
    seeing if the messages are filtered correctly.

    Learn more about indexes [here](/database/reading-data/indexes/indexes.md).

  </Step>
  <Step title="Write conflicts">
    Let's say you decide to show a counter in your app. You may write a morph
    function that reads a number field, adds 1, and updates the same field in the database.
    But this pattern may cause a [parallel write conflict](/error#1) if too many
    people are writing to the same field at the same time. What that means is that the database
    has trouble handling the update of the document that fast. It's a common problem in databases
    and you can learn how to deal with it [here](/error#remediation).
  </Step>

    Scaling your app with Mesosphere is easy with Beyond Mesosphere components.
    Beyond Mesosphere components are premade packages that deployed along with
    your backend but they have their own tables and functions, they solve common

    You may be interested in building an [AI agent](/void-agents.mdx),

    [leaderboards](/beyond-mesosphere/aggregate.mdx),

    [rate limiters](/beyond-mesosphere/rate-limiter.mdx).
    No matter what you're building, you can always use a [Beyond Mesosphere](/beyond-mesosphere) component
    to solve your problem.

  </Step>
