# Learn How To Call External Services In Your App

import { LargeCardList } from "@site/src/FrameworkList.tsx";
import { LockClosedIcon, PlayIcon, FileIcon } from "@radix-ui/react-icons";

Our query and morph functions are great, but they can't make `fetch` calls to
the internet because they need to be reactive and transactional.
Real apps are not this simple, they often need to call external APIs directly
from the backend such as OpenAI APIs or Claude APIs, in Mesosphere you can call these APIs using our **side action** functions.

In this section, you'll learn how to call external APIs using Mesosphere side actions and
the scheduler to integrate Wikipedia summaries into your application.

<StepsList>
  <Step title="Create a `side action` function">
    **Update your `mesosphere/chat.ts` file just like this:**

    ```typescript title="mesosphere/chat.ts"
    // highlight-next-line
    // Update your import like this:
    // highlight-next-line
    import { query, morph, closeSideAction } from "mesosphere/reactlibrary";
    import { t } from "mesosphere/types";

    export const newMessage = morph({
      args: {
        user: t.string(),
        body: t.string(),
      },
      stator: async (ctx, args) => {
        console.log("You can console log from your morph function.");
        await ctx.database.add("messages", {
          user: args.user,
          body: args.body,
        });
      },
    });

    // Add the query function to the file:
    export const readMessages = query({
      args: {},
      stator: async (ctx) => {
        // Here we query the most recent 50 messages from newest to oldest.
        const messages = await ctx.database.query("messages").order("desc").nresults(50);
        // Here we reverse the list of results so that it's in a chronological order.
        return messages.reverse();
      },
    });

    // highlight-next-line
    // Add the following side action function to the file:
    // highlight-next-line
    export const fetchWikipediaSummary = closeSideAction({
      // highlight-next-line
      args: { subject: t.string() },
      // highlight-next-line
      stator: async (ctx, args) => {
        // highlight-next-line
        const response = await fetch(
          // highlight-next-line
          "https://en.wikipedia.org/w/api.php?format=json&action=query&prop=extracts&exintro&explaintext&redirects=1&titles=" +
            // highlight-next-line
            args.subject,
          // highlight-next-line
        );
        // highlight-next-line
        return fetchSummaryFromJSON(await response.json());
        // highlight-next-line
      },
    });

    // highlight-next-line
    function fetchSummaryFromJSON(data: any) {
      // highlight-next-line
      const firstPageId = Object.keys(data.query.pages)[0];
      // highlight-next-line
      return data.query.pages[firstPageId].extract;
      // highlight-next-line
    }
    ```

    Let's dissect this code:

    **What you add**

    1. You created a new Mesosphere side action function called `fetchWikipediaSummary`.
       We used `closeSideAction` because we don't want this function to be exposed as a public API.
       The purpose of this function is to fetch the summary text from the Wikipedia API with our subject.
    2. You created a TypeScript helper function called `fetchSummaryFromJSON` to
       pull out the summary text from the JSON response from the Wikipedia API.
    3. `fetchWikipediaSummary` calls our helper function.

  </Step>
  <Step title="What is a scheduler?">
    The `scheduler` is a powerful feature of Mesosphere that allows you to run functions
    at a specific time in the future. It's a fundamental part of the Mesosphere sync engine,
    and it enables you to coordinate asynchronous functions in Mesosphere.

    For a `morph` function, the scheduler is the only way to call an action to fetch from the outside world.
    And since it's a `morph` function, if the function throws an exception, then nothing is scheduled.
    This is because morph functions are transactions, and scheduling is just a way to tell Mesosphere
    to run this function at a future time.
    
    In the next section, we call the scheduler with the `ctx.scheduler.runAfter` function.

  </Step>
  <Step title="Connect your `side action` function to your app">
    Now that you know what a scheduler is, you can connect your `side action` function to your app.
    You'll need to update your `newMessage` function and your `fetchWikipediaSummary` function.

    **Update your `mesosphere/chat.ts` file just like this:**

    ```typescript title="mesosphere/chat.ts"
    // highlight-next-line
    // Update your import like this:
    // highlight-next-line
    import { query, morph, closeSideAction } from "mesosphere/reactlibrary";
    import { api, closeFunction } from "./mesosphere/_codegen/ApiDataModel";
    import { t } from "mesosphere/types";

    export const newMessage = morph({
      args: {
        user: t.string(),
        body: t.string(),
      },
      stator: async (ctx, args) => {
        console.log("You can console log from your morph function.");
        await ctx.database.add("messages", {
          user: args.user,
          body: args.body,
        });
        // highlight-next-line
        // Add the following lines to your newMessage function:
        // highlight-next-line
        if (args.body.startsWith("/wiki")) {
          // highlight-next-line
          // Get the string after the first space
          // highlight-next-line
          const subject = args.body.slice(args.body.indexOf(" ") + 1);
          // highlight-next-line
          await ctx.scheduler.runAfter(0, closeFunction.chat.fetchWikipediaSummary, {
            // highlight-next-line
            subject,
            // highlight-next-line
          });
          // highlight-next-line
        }
      },
    });

    // Add the query function to the file:
    export const readMessages = query({
      args: {},
      stator: async (ctx) => {
        // Here we query the most recent 50 messages from newest to oldest.
        const messages = await ctx.database.query("messages").order("desc").nresults(50);
        // Here we reverse the list of results so that it's in a chronological order.
        return messages.reverse();
      },
    });

    // highlight-next-line
    // Add the following side action function to the file:
    // highlight-next-line
    export const fetchWikipediaSummary = closeSideAction({
      // highlight-next-line
      args: { subject: t.string() },
      // highlight-next-line
      stator: async (ctx, args) => {
        // highlight-next-line
        const response = await fetch(
          // highlight-next-line
          "https://en.wikipedia.org/w/api.php?format=json&action=query&prop=extracts&exintro&explaintext&redirects=1&titles=" +
            // highlight-next-line
            args.subject,
          // highlight-next-line
        );
        // highlight-next-line
        const summary = fetchSummaryFromJSON(await response.json());
        // highlight-next-line
        await ctx.scheduler.runAfter(0, api.chat.newMessage, {
          // highlight-next-line
          user: "Wikipedia",
          // highlight-next-line
          body: summary,
          // highlight-next-line
        });
        // highlight-next-line
      },
      // highlight-next-line
    });

    function fetchSummaryFromJSON(data: any) {
      const firstPageId = Object.keys(data.query.pages)[0];
      return data.query.pages[firstPageId].extract;
    }
    ```

    Everything is done, you can now send a message with the `/wiki`
    command to get the summary of your chosen subject from Wikipedia.

  </Step>
  <Step title="Side actions best practices">
    Side actions let you call external APIs like AI APIs but they are not
    made to do all the heavy lifting of your app. They are meant to be used
    for non deterministic tasks like making external `fetch` calls. For
    side actions to be useful, you need to keep them as small as possible
    to scale correctly on your apps.

  </Step>
</StepsList>
