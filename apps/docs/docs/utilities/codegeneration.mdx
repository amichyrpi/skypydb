title: "Code Generation"
icon: "square-code"
---

Mesosphere automatically generates code when you run `npx mesosphere dev`. It creates the following files:

- `mesosphere/deploy`: a utility that exposes a public `api` generated from your functions. You can use it in your app to call your functions like this:

```typescript
import { callwrite } from "mesosphere/reactlibrary";

// Import the generated API from the deploy file
import { api } from "../mesosphere/deploy";

// Call your function using the API
callwrite(api.yourfunction.yourwritefunction);
```

- `mesosphere/README.md`: a file containing basic information on how to use Mesosphere. This is not a guide for AI agents. If you're interested in using AI with Mesosphere, check out the [AI Agents](./aiagents/quickstart) guide.

- `mesosphere/tsconfig.json`: this file contains the TypeScript compiler options used to type-check your functions. You can modify some parameters in this file. Example:

```json mesosphere/tsconfig.json
{
  /* This TypeScript project config describes the environment that
   * Mesosphere functions run in and is used to type-check them.
   * You can modify it, but some settings are required to use Mesosphere.
   */
  "compilerOptions": {
    /* These settings can be modified. */
    "allowJs": true,
    "strict": true,
    "moduleResolution": "Bundler",
    "jsx": "react-jsx",
    "skipLibCheck": true,
    "allowSyntheticDefaultImports": true,

    /* These compiler options cannot be modified. */
    "target": "ESNext",
    "lib": ["ES2021", "dom"],
    "forceConsistentCasingInFileNames": true,
    "module": "ESNext",
    "isolatedModules": true,
    "noEmit": true
  },
  "include": ["./**/*"]
}
```

- `.env`: this file contains your credentials to use Mesosphere. It should not be committed. Example:

```text .env
# Automatically generated with the `npx mesosphere dev` command

MESOSPHERE_URL="your_mesosphere_url"

MESOSPHERE_API_KEY="your_api_key"
```
