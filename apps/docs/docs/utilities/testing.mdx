In Mesosphere, testing your functions is straightforward. Using the [Mesosphere CLI](../cli/quickstart), you can directly test your functions before deploying them to production.

When you run `npx mesosphere dev`, your functions are deployed to the server in development mode. If an error occurs, a detailed message is returned, including the line number and the reason for the failure.

Functions will not be deployed until they compile successfully.

# Example

Here we will intentionally trigger a failure:

```typescript mesosphere/users.ts
import { writeFunction } from "mesosphere/reactlibrary";
import { type } from "mesosphere/type";

export const newuser = writeFunction({
  args: {
    name: type.string(),
    email: type.string(),
  },
  handler: async (mesosphere, args) => {
    // Here we trigger an error by using `get` instead of `add`
    const newuserID = await mesosphere.database.get("users", {
      name: args.name,
      email: args.email,
    });
    return newuserID;
  },
});
```

The CLI will throw an error indicating incorrect usage of the `get` method at line 11 in the `users` file. Did you mean `add`?
